## ALLOC — Asset Allocation / Glide Path

จุดประสงค์: จัดกรอบ "ควรเสี่ยงแค่ไหนในแต่ละช่วงชีวิต" — ไม่ใช่สั่ง % ตายตัว เพราะ % ที่เหมาะขึ้นกับสถานการณ์เฉพาะตัวที่ผู้ใช้รู้ดีกว่า

### หลักการ glide path
ความสามารถรับความเสี่ยงควรลดลงเมื่อใกล้และเข้าสู่วัยเกษียณ เพราะเวลาฟื้นตัวจาก drawdown สั้นลง:

| ช่วง | บทบาทพอร์ต | โน้ต |
|---|---|---|
| **Accumulation (ยังห่างเกษียณ >10–15 ปี)** | เน้นเติบโต ทนผันผวนได้ เพราะมีเวลาฟื้น | ความเสี่ยงใหญ่คือ "โตไม่ทันเงินเฟ้อ" ไม่ใช่ความผันผวนระยะสั้น |
| **Transition (ใกล้เกษียณ ~5–10 ปี)** | เริ่ม de-risk ลดส่วนผันผวนสูง | ช่วงนี้เปราะต่อ sequence risk มากขึ้น |
| **Decumulation (อยู่ในวัยเกษียณ)** | สมดุล: ส่วนเติบโตสู้ longevity + ส่วนมั่นคง/สภาพคล่องไว้ถอน | อย่า de-risk จนโตไม่ทันเงินเฟ้อตลอด 20–30 ปี |

### Risk capacity vs tolerance (ใช้ตัวต่ำกว่าเป็นกรอบ)
- **Capacity (เชิงวัตถุ):** เวลาถึงเกษียณ, เงินสำรอง, ความมั่นคงรายได้, รายได้หลังเกษียณอื่นที่การันตี (บำนาญ), ภาระพึ่งพิง
- **Tolerance (เชิงใจ):** ทนเห็นพอร์ตติดลบกี่ % โดยไม่เทขาย
- ถ้า capacity สูงแต่ tolerance ต่ำ → กรอบที่เหมาะคือฝั่ง tolerance (พอร์ตที่ถือไม่ไหวคือพอร์ตที่จะถูกเทตอนแย่สุด)

### สิ่งที่ทำได้ / ไม่ทำ
- ✅ อธิบาย trade-off growth vs stability, เสนอ **ช่วง** (band) ตามช่วงชีวิต, อธิบาย rebalancing, ชี้ความเสี่ยง concentration (เช่น พึ่งหุ้นนายจ้างตัวเดียว)
- ✅ วิจารณ์ rule-of-thumb หยาบ ๆ (เช่น "100 − อายุ = %หุ้น") ว่าใช้เป็นจุดตั้งต้นได้แต่ไม่ได้คิดถึง capacity/รายได้อื่น
- ❌ ฟันธง "ใส่หุ้น X% ตราสารหนี้ Y%" เป็นคำสั่งเฉพาะบุคคล → เสนอเป็นกรอบ + เงื่อนไข แล้วคืนการเลือกให้ผู้ใช้/นักวางแผน
- ❌ เลือกหุ้น/กองทุนรายตัว → นั่นคือ `sgui-stock-analysis`

### Output contract (ALLOC)
1. ช่วงชีวิตของผู้ใช้ + ความเสี่ยงหลักของช่วงนั้น
2. capacity vs tolerance — ตัวไหนเป็นกรอบ
3. กรอบ allocation เป็น **ช่วง** + เหตุผล (ไม่ใช่ % ฟันธง)
4. **Bucket map** (ถ้าเข้าข่าย — ดู Bucket Layer ล่าง)
5. rebalancing + concentration ที่ต้องระวัง
6. ส่งต่อ VEH (ใส่ในเครื่องมือไหน) / DEC

### Bucket Layer (reusable — เรียกเมื่อเข้าข่าย ไม่บังคับทุกครั้ง)
แยกเงินตามหน้าที่/เวลา/ความเสี่ยง map สินทรัพย์ + เงินออมอนาคตเข้าถัง **โดยไม่ระบุผลิตภัณฑ์เฉพาะ** เว้นผู้ใช้ระบุเอง:

| ถัง | หน้าที่ | เรียกเมื่อ |
|---|---|---|
| **Safety** | เงินสำรอง + รายจ่ายระยะใกล้ | cashflow / ใกล้เกษียณ |
| **Bridge** | cover ช่วงก่อนรายได้ (บำนาญ/ประกันสังคม/RMF/annuity) เริ่ม | **บังคับเมื่ออายุเกษียณ < อายุเริ่มรับรายได้** (ดู bridge math ใน projection.md) |
| **Income** | สินทรัพย์/กระแสเงินสำหรับช่วงเกษียณต้น ๆ | DRAW / ใกล้เกษียณ |
| **Growth** | ระยะยาว สู้เงินเฟ้อ + longevity | ALLOC / horizon ยาว |
| **Protection** | ประกัน, สำรองค่ารักษา, family/estate risk | RSK / สุขภาพ / ภาระครอบครัว |

> ไม่ต้องโชว์ครบ 5 ถังทุกครั้ง — ใช้เฉพาะถังที่เกี่ยวกับคำถาม · **Bridge เป็นถังบังคับ**เมื่อเกษียณก่อนรายได้เริ่ม (ลืม = ตรงกับ critical-fail เรื่อง bridge)

## VEH — Vehicles & สิทธิภาษี (Thai-first, global-friendly)

จุดประสงค์: จับคู่ "เงินที่จะออม-ลงทุน" กับ "ภาชนะ" ที่เหมาะ โดยดูโครงสร้าง: สิทธิภาษี, lock-up/สภาพคล่อง, เหมาะกับใคร

> **กฎเหล็ก:** ไม่ฝังวงเงินลดหย่อน/อัตรา/เงื่อนไขปีล่าสุดเป็น fact (เปลี่ยนบ่อย) ให้พูดเชิงโครงสร้าง แล้วบอกผู้ใช้ยืนยันกับแหล่งทางการ: **กรมสรรพากร (rd.go.th), สำนักงานประกันสังคม (sso.go.th), กบข (gpf.or.th), กอช (nsf.or.th), และ prospectus/บลจ.** ก่อนใช้จริง

### เครื่องมือไทย (เชิงโครงสร้าง)
| เครื่องมือ | ลักษณะ | จุดต้องเช็กล่าสุด |
|---|---|---|
| **กองทุนรวมเพื่อเกษียณ (เช่น RMF)** | ลงทุนระยะยาว มีสิทธิลดหย่อน มีเงื่อนไขถือ/อายุก่อนขายได้ | วงเงินลดหย่อน, เงื่อนไขอายุ/ปีถือ, เพดานรวมกับ vehicle อื่น |
| **กองทุนรวมลดหย่อนระยะยาวอื่น (เช่น SSF / Thai ESG)** | ลดหย่อน + เงื่อนไขถือ ต่างจาก RMF | สิทธิ/เพดาน/เงื่อนไขถือปีปัจจุบัน (เปลี่ยนบ่อย) |
| **กองทุนสำรองเลี้ยงชีพ (PVD)** | นายจ้างสมทบ (employer match = เงินที่ไม่ควรพลาด) + ลดหย่อน | สัดส่วน match, นโยบายการลงทุนที่เลือกได้, เงื่อนไขเมื่อออกงาน |
| **ประกันสังคม ม.33/39/40** | บำนาญ/บำเหน็จชราภาพ ฐานรองรับขั้นต่ำ | สิทธิที่จะได้ตามมาตราของผู้ใช้ (ให้เช็ก สปส.) |
| **กบข** (ข้าราชการ) | กองทุนบำเหน็จบำนาญข้าราชการ + เลือกแผนลงทุนได้ | แผนที่เลือก, สิทธิเมื่อเกษียณ |
| **กอช** (อาชีพอิสระ) | ออมพร้อมรัฐสมทบ สำหรับคนไม่มีนายจ้าง | เงื่อนไขสมทบ/บำนาญ |
| **ประกันบำนาญ (annuity)** | รายได้ตลอดชีพ ลด longevity risk + อาจลดหย่อน | ผลตอบแทน/เงื่อนไข/ค่าธรรมเนียม — อ่าน policy ละเอียด |
| **เงินลงทุนทั่วไป (taxable)** | ยืดหยุ่น สภาพคล่องสูง ไม่มีสิทธิพิเศษ/ไม่มี lock | ใช้เสริมเมื่อ vehicle สิทธิภาษีเต็มเพดานแล้ว |

### ถ้าบริบทต่างประเทศ — map เข้าโครงเดียวกัน
tax-advantaged retirement account (เช่น 401(k)/IRA/Roth สหรัฐฯ, pension/ISA สหราชอาณาจักร) ↔ "เครื่องมือสิทธิภาษี + lock", employer match ↔ PVD match, taxable brokerage ↔ เงินลงทุนทั่วไป — ใช้หลักจัดลำดับเดียวกัน ผู้ใช้ยืนยันกฎประเทศนั้น

### ลำดับจัดสรรที่นิยม (เป็นกรอบ ไม่ใช่คำสั่ง)
1. ใช้ **employer match ให้เต็มก่อน** (เงินสมทบฟรี ผลตอบแทนทันทีสูงสุด)
2. เคลียร์หนี้ดอกเบี้ยสูง / มีเงินสำรองพอ (ดู BASE)
3. ใช้ **เครื่องมือสิทธิภาษี** ตามเพดานที่เหมาะกับสถานการณ์ภาษีของผู้ใช้
4. ส่วนเกิน → เงินลงทุนทั่วไป (taxable) เพื่อความยืดหยุ่น

> เตือนเสมอ: **อย่าให้หางภาษีกระดิกหมา** — สิทธิลดหย่อนเป็นโบนัส ไม่ใช่เหตุผลเดียวที่ซื้อ ดูความเหมาะกับเป้า/สภาพคล่อง/เงื่อนไขถือด้วย และระวัง lock-up ที่ทำให้เงินออกไม่ได้ตอนจำเป็น

### Output contract (VEH)
1. จับคู่เงินออม → เครื่องมือ (เชิงโครงสร้าง) + เหมาะกับใคร/เพราะอะไร
2. ธงสิทธิภาษี/เงื่อนไข lock ที่ **ผู้ใช้ต้องไปยืนยันล่าสุด** (ชี้แหล่งทางการ)
3. employer match ที่ยังไม่ได้ใช้เต็ม (ถ้ามี) — flag เด่น
4. เตือนเรื่อง lock-up / สภาพคล่อง
5. ส่งต่อ DEC

## Anti-patterns (ALLOC / VEH)
- ฟันธง % allocation เฉพาะบุคคลเหมือนเป็นคำสั่ง
- de-risk เป็นเงินสด/ตราสารหนี้ล้วนตอนเกษียณ จนโตไม่ทันเงินเฟ้อ 20–30 ปี
- มองข้าม risk tolerance จริงของผู้ใช้ (ออกแบบพอร์ตที่ถือไม่ไหว)
- ฝังวงเงินลดหย่อน/เงื่อนไขปีล่าสุดเป็น fact
- ซื้อ vehicle เพราะลดหย่อนอย่างเดียว โดยไม่ดู lock-up/ความเหมาะ
- ลืม employer match (เงินฟรีที่พลาด)
