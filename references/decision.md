# DEC — Decision Layer (กรอบตัดสินใจเชิงแผน)

ใช้เมื่อผู้ใช้ถามเชิง action: ออมเพิ่มดีไหม, เกษียณเร็วได้ไหม, ปรับ allocation, เปลี่ยน vehicle, ลด/เพิ่มรายจ่ายเป้า ทำงานภายใต้ Operating Rules — โดยเฉพาะ **กฎ 6 (ไม่ฟันธงเฉพาะบุคคล → conditional)** และ **critical-fail: ไม่มี risk checklist / ไม่มี decision journal = ผลใช้ไม่ได้**

**Input ขั้นต่ำ:** เป้า (อายุเกษียณ/รายจ่ายเป้า) + การตัดสินใจที่กำลังชั่ง
**ต้องมีเพื่อคุณภาพสูง:** ฐานะปัจจุบัน (BASE), Number/gap (NUM/GAP), risk capacity + tolerance, รายได้หลังเกษียณอื่น, สมมติฐานที่ยืนยันแล้ว
**ถ้าขาดข้อมูลแผน** → ทำได้แค่ **conditional framework** บอกตรง ๆ ว่า "สรุปแผนเฉพาะตัวไม่ได้จากข้อมูลปัจจุบัน" ห้ามฟันธง

---

## 1. Frame เป็น trade-off ไม่ใช่คำตอบเดียว

ทุกการตัดสินใจเกษียณคือการแลกระหว่าง lever ที่ผู้ใช้ควบคุมได้ — แสดงให้ครบก่อนชี้ทาง:

| Lever | ขยับแล้วเกิดอะไร | ต้นทุน/ความเสี่ยง |
|---|---|---|
| ออม/ลงทุนเพิ่ม | ลด shortfall | กระทบคุณภาพชีวิตวันนี้ |
| เลื่อนเกษียณออก | ออมต่อ + ถอนสั้นลง (ขยับ Number เยอะ) | เวลาเกษียณน้อยลง สุขภาพ/ความตั้งใจ |
| ลดรายจ่ายเป้าหลังเกษียณ | ลด Number ตรง ๆ | มาตรฐานชีวิตหลังเกษียณ |
| รับความเสี่ยงพอร์ตมากขึ้น | อาจโตเร็วขึ้น | sequence/drawdown risk สูงขึ้น — พึ่ง lever นี้มากสุดเปราะสุด |
| หารายได้หลังเกษียณ (part-time/ค่าเช่า) | ลดภาระพอร์ต | ความแน่นอน/สุขภาพ/ความเต็มใจ |

> เตือน: ระวังการแก้ปัญหา shortfall ด้วยการ "เพิ่มสมมติฐานผลตอบแทน" — นั่นคือเปลี่ยนความหวัง ไม่ใช่เปลี่ยนแผน

## 2. Conditional Decision Framework

ตอบเป็น path ตามเงื่อนไข ไม่ใช่คำสั่ง — แต่ละ path มีเงื่อนไข + เหตุผล + ความเสี่ยง:

| Path | เงื่อนไขที่ทำให้สมเหตุผล | ความเสี่ยงของ path นี้ |
|---|---|---|
| เดินแผนเดิมต่อ | on track ใน base + conservative ยังรับได้ | … |
| ออมเพิ่ม X | shortfall ปานกลาง + มี room ในงบวันนี้ | กดคุณภาพชีวิตปัจจุบัน |
| เลื่อนเกษียณ | shortfall สูง + ยังทำงานไหว | … |
| ลดรายจ่ายเป้า | รายจ่ายเป้าสูงเกินจำเป็น | … |
| ปรับ allocation/vehicle | ปัญหาอยู่ที่โครงสร้าง ไม่ใช่ยอดออม | sequence/lock risk |

ครอบคลุมเงื่อนไข: gap (3 scenario), risk capacity/tolerance, สภาพคล่อง, ความครบข้อมูล, review trigger

## 3. Behavioral Bias Check (ทำก่อนสรุปทุกครั้ง)

| Bias | คำถามวินิจฉัย |
|---|---|
| Optimism / planning fallacy | ใช้สมมติฐานผลตอบแทน/รายจ่ายที่เข้าข้างตัวเองไหม |
| Present bias | ให้น้ำหนักความสุขวันนี้จนละเลยตัวเองในอนาคตไหม (หรือกลับกัน—ประหยัดเกินจนไม่ได้ใช้ชีวิต) |
| Anchoring | ติดกับ "ตัวเลขในใจ" (เช่นต้องมี 10 ล้าน) โดยไม่อิงรายจ่ายจริงไหม |
| Overconfidence | มั่นใจผลตอบแทน/อายุงาน/สุขภาพเกินหลักฐานไหม |
| Recency | ตัดสินจากผลตลาด/ข่าวช่วงนี้มากไปไหม |
| Loss aversion | กลัวเสี่ยงจน de-risk เกินไปจนโตไม่ทันเงินเฟ้อไหม |
| Herding | ทำตามคนอื่น/เทรนด์ (เช่น FIRE) โดยไม่ตรงบริบทตัวเองไหม |

**กฎเหล็ก:** ห้ามใช้ "เพื่อนทำแบบนี้" หรือ "เทรนด์กำลังมา" เป็นเหตุผลหลัก · ระวังทั้งประหยัดเกินและใช้เกิน — สองด้านของ present bias

## 4. Review / Re-plan Triggers (อย่างน้อย 5)
รายได้/รายจ่ายเปลี่ยนมีนัย · ตลาด drawdown ใหญ่ (ทบทวน sequence risk) · กฎ/ภาษี vehicle เปลี่ยน · เป้าชีวิตเปลี่ยน (เกษียณเร็ว/ช้า, ภาระครอบครัว) · ใกล้เกษียณ ~5–10 ปี (สลับเข้าโหมด DRAW/de-risk) · ทบทวนตามเวลา (อย่างน้อยปีละครั้ง)

## 5. Action Plan (แปลงเป็นสิ่งที่ทำได้จริง — framed as options)
ปิดด้วย next action ที่สังเกตได้ แบ่งตาม horizon **เป็นทางเลือก ไม่ใช่คำสั่ง** (ยกเว้น housekeeping พื้นฐาน เช่น ตั้งเงินสำรอง/เคลียร์หนี้ดอกสูง):

| Horizon | ตัวอย่างสิ่งที่ทำได้ |
|---|---|
| **30 วัน** | ยืนยันสมมติฐานที่ยังไม่ verified, เช็กสิทธิประกันสังคม/PVD match, ตั้งเงินสำรองให้ครบ |
| **3–6 เดือน** | ทบทวน cashflow priority, ปรับอัตราออม/allocation ตาม lever ที่เลือก, ยืนยันกฎภาษี vehicle ล่าสุด |
| **Annual review** | rerun NUM/GAP ด้วยตัวเลขจริงปีล่าสุด, เช็ก trigger, ใกล้เกษียณ ~10 ปีสลับเข้า DRAW/de-risk |

ทุก action ควร reversible เท่าที่ทำได้ และผูกกับ trigger/falsifiability ในข้อ 4 และ 6

## 6. Decision Journal (ปิดท้ายทุก output ที่เป็น decision)

| ช่อง | กรอก |
|---|---|
| วันที่ / การตัดสินใจที่ชั่ง / horizon | … |
| สถานะแผนตอนนี้ (gap 3 scenario สั้น ๆ) | … |
| สมมติฐานหลักที่พึ่ง (return/เงินเฟ้อ/อายุขัย) + ยืนยันแล้วหรือยัง | … |
| Lever ที่เลือกขยับ + trade-off ที่ยอมรับ | … |
| Conditional paths (สรุป) | … |
| **อะไรจะทำให้รู้ว่าแผนหลุด** (falsifiability) | เช่น "ถ้า conservative scenario แตะ shortfall >X% ที่อายุ Y" |
| Review/re-plan trigger | … |
| Missing data / สมมติฐานที่ยังไม่ยืนยัน | … |
| Final status | แผนพอใช้ตัดสินใจได้ / ต้องข้อมูลเพิ่ม / ควรปรึกษานักวางแผนมีใบอนุญาต |

## Output contract (DEC)
1. Decision context + ข้อจำกัด (สรุปเฉพาะตัวได้แค่ไหน)
2. สถานะแผน (gap 3 scenario) + สมมติฐานที่พึ่ง
3. ตาราง trade-off lever
4. Conditional decision paths
5. Bias check
6. Review/re-plan triggers
7. **Action plan (30d / 3–6mo / annual)** — framed as options
8. **Decision journal**
9. Bottom line แบบ conditional ≤5 bullet + ความมั่นใจ (band)

**Next step:** ปิดด้วยเตือนว่าคนตัดสินใจสุดท้ายและความเสี่ยงคือผู้ใช้ + ถ้าเป็นเรื่องภาษี/ผลิตภัณฑ์เฉพาะตัว ให้ชี้ปรึกษานักวางแผนมีใบอนุญาต (CFP)

## Anti-patterns (DEC)
- ฟันธง "ออมเดือนละ X / เกษียณได้เลย" เป็นคำสั่งเฉพาะบุคคล
- แก้ shortfall ด้วยการเพิ่มสมมติฐานผลตอบแทน (เปลี่ยนความหวังไม่ใช่แผน)
- เสนอ lever เดียว (ออมเพิ่ม) โดยไม่โชว์ trade-off อื่น
- ข้าม bias check / decision journal
- เชียร์/ห้าม FIRE ตามเทรนด์โดยไม่ดูบริบทเฉพาะตัว
- ไม่มี falsifiability — ผู้ใช้เลยไม่รู้ว่าเมื่อไรต้องทบทวน
