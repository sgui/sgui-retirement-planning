# Decumulation — DRAW

ใช้เมื่อใกล้/อยู่ในวัยเกษียณ: "ถอนเงินยังไงไม่ให้หมดก่อน / เงินจะพอถึงกี่ขวบ" ทำงานภายใต้ Operating Rules — โดยเฉพาะ **ต้องพูดถึง sequence-of-returns เสมอ (ข้าม = critical-fail)** และ **3 scenario + แยก real/nominal**

การถอน (decumulation) ยากกว่าการสะสม เพราะลำดับผลตอบแทนสำคัญ ไม่ใช่แค่ค่าเฉลี่ย

---

## 1. Sequence-of-returns risk (แกนของ DRAW)

ช่วงสะสม ลำดับปีดี/ปีแย่ไม่สำคัญ (ค่าเฉลี่ยพอ) แต่ช่วงถอน **ถ้าตลาดตกหนักช่วง 2–3 ปีแรกหลังเกษียณ ขณะที่ยังถอนเงินออกอยู่ เงินต้นถูกกินเร็วจนฟื้นไม่ทัน** แม้ค่าเฉลี่ยระยะยาวจะเท่ากัน ผลลัพธ์ต่างกันมาก

ทางลดที่ต้องพูดถึง:
- **Cash/bond buffer** กันรายจ่าย ~1–3 ปีไว้ในสินทรัพย์มั่นคง จะได้ไม่ต้องขายของที่กำลังตก
- **ถอนแบบยืดหยุ่น** ลดการถอนปีที่ตลาดแย่ (ดู guardrails ข้อ 3)
- **เริ่มถอนต่ำหน่อยในปีแรก ๆ** เผื่อ buffer
- อย่าเข้าเกษียณด้วยพอร์ตเสี่ยงสูงล้วน (ดู ALLOC transition)

---

## 2. Safe Withdrawal Rate (SWR) — framework ไม่ใช่กฎตายตัว

"กฎ 4%" มาจากงานวิจัยตลาดสหรัฐฯ บนสมมติฐานเฉพาะ (เช่น ขอบเขต ~30 ปี, สัดส่วนพอร์ต/ตลาดในอดีตแบบหนึ่ง) **ใช้เป็นจุดตั้งต้นทางความคิดได้ แต่ต้องบอกข้อจำกัด:**
- ขอบเขตเกษียณยาวกว่า 30 ปี (เกษียณเร็ว/อายุยืน) → SWR ที่ปลอดภัยต่ำลง
- ผลตอบแทน/เงินเฟ้อในอนาคต ≠ อดีต
- ไม่ได้คิดภาษี/ค่าธรรมเนียม/รายได้อื่น
- ตลาด/สินทรัพย์ของผู้ใช้อาจต่างจากสมมติฐานต้นทาง

→ ใช้ SWR เป็น "ช่วงเริ่มต้น" แล้วเทียบกับ depletion model ของผู้ใช้เอง อย่าเสนอเป็นตัวเลขศักดิ์สิทธิ์

---

## 3. กลยุทธ์ถอน (เสนอเป็นทางเลือก ไม่ฟันธง)

| กลยุทธ์ | วิธี | เหมาะเมื่อ |
|---|---|---|
| **Fixed real** | ถอนยอดคงที่ (ปรับเงินเฟ้อ) ทุกปี | อยากคาดเดารายรับได้ แต่เปราะต่อ sequence risk |
| **Dynamic / guardrails** | ลดถอนปีตลาดแย่ เพิ่มได้ปีตลาดดี | รับความผันผวนรายรับได้ ยืดอายุพอร์ตได้ดีกว่า |
| **Floor-and-upside** | แยก "รายจ่ายจำเป็น" ให้แหล่งมั่นคง/ตลอดชีพ (บำนาญ/annuity) ส่วน "รายจ่ายไม่จำเป็น" มาจากพอร์ตเสี่ยง | อยากการันตีขั้นต่ำของชีวิต |
| **Bucket** | แบ่งเงินเป็นถัง: เงินสด (1–3 ปี) / ตราสารหนี้ (กลาง) / เติบโต (ยาว) เติมถังจากถังยาวเมื่อตลาดเอื้อ | จัดการ sequence risk เชิงพฤติกรรมง่าย |

---

## 4. ลำดับการถอนข้ามบัญชี (Thai-first, ยืนยันกฎล่าสุด)

ลำดับถอนมีผลต่อภาษีและเงื่อนไข lock — เชิงโครงสร้าง:
- บัญชี/กองทุนสิทธิภาษีมักมี **เงื่อนไขอายุ/ปีถือก่อนถอนได้โดยไม่เสียสิทธิ** (เช่น RMF) → ต้องวางลำดับไม่ให้ผิดเงื่อนไข
- **Bridge ก่อน:** ถ้าเกษียณก่อนรายได้/บัญชีสิทธิภาษีเริ่มจ่ายได้ ช่วง bridge ต้องถอนจาก **Safety/Bridge/taxable** ที่เข้าถึงได้ ไม่ใช่บัญชีที่ยังติดเงื่อนไข — map กับ income-start schedule (ดู projection.md สูตร 6)
- ประสานกับ **รายได้ตลอดชีพ** (ประกันสังคม/กบข/บำนาญ/annuity) ว่าจะเริ่มรับเมื่อไร ส่วนไหนให้พอร์ต cover
- **ยืนยันเงื่อนไข/ภาษีล่าสุดกับแหล่งทางการเสมอ** (สรรพากร/สปส./กบข/บลจ.) — ไม่ฝังกฎที่อาจล้าสมัย

## 5. Depletion math (ใช้ตรวจว่าเงินพอถึงกี่ขวบ)

ถามว่า "เงินก้อน `P` ถอนปีละ `W` (real) ที่ผลตอบแทนจริง `rr` จะพอกี่ปี":
- ปีที่เงินหมด ≈ แก้หาจาก `P = W × [1 − (1+rr)^(−m)] / rr`
- ถ้า `W/P` ≤ ~`rr` → ตามทฤษฎีถอนได้แทบไม่จำกัด (กินจากผลตอบแทน) แต่ **ต้องโชว์ผลแบบ 3 scenario** เพราะ `rr` ไม่แน่นอน และ **เตือน sequence risk** ว่าโมเดลค่าเฉลี่ยนี้ไม่ได้สะท้อนความเสี่ยงปีแย่ช่วงต้น
- เสมอ: หัก/บวกรายได้อื่นก่อนใส่ `W`

---

## Output contract (DRAW)
1. **Assumption Ledger** + ระบุ real/nominal
2. ผล "เงินพอถึงอายุเท่าไร" หรือ "ถอนได้ปีละเท่าไรอย่างปลอดภัย" เป็น **3 scenario** `[คำนวณ]`
3. **Sequence-of-returns** — อธิบายผล + ทางลด (บังคับ)
4. กลยุทธ์ถอน 2–3 ทางเลือก + trade-off (ไม่ฟันธง)
5. ลำดับถอนข้ามบัญชี + ธงเงื่อนไข/ภาษีที่ต้องยืนยันล่าสุด
6. ความมั่นใจ (band) + ส่งต่อ DEC/RSK

## Anti-patterns (DRAW)
- ใช้ "กฎ 4%" เป็นตัวเลขศักดิ์สิทธิ์โดยไม่บอกข้อจำกัด/ขอบเขตเวลา
- ข้าม sequence-of-returns (critical-fail)
- คำนวณด้วยผลตอบแทนเฉลี่ยตัวเดียว ไม่ทำ scenario
- ลืมรายได้ตลอดชีพ (บำนาญ/annuity) ที่ลดภาระพอร์ต
- แนะลำดับถอนโดยฝังกฎภาษี/เงื่อนไขที่อาจล้าสมัยเป็น fact
- เสนอกลยุทธ์เดียวแบบฟันธง แทนทางเลือก + trade-off
