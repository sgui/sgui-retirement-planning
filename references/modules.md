## BASE — สแกนฐานะตั้งต้น (ทำก่อนวางแผนเสมอ)

จุดประสงค์: ตอบ "ตอนนี้อยู่ตรงไหน" ก่อนคำนวณว่า "ต้องไปถึงไหน" ถ้าข้ามขั้นนี้ NUM/GAP จะลอย

**Input ขั้นต่ำ:** อายุ + สินทรัพย์เพื่อเกษียณปัจจุบัน + รายได้/รายจ่ายต่อเดือน
**ต้องมีเพื่อคุณภาพสูง:** หนี้ (คงเหลือ/อัตราดอกเบี้ย), เงินสำรองฉุกเฉิน, เงินออม-ลงทุนต่อเดือนปัจจุบัน, สวัสดิการนายจ้าง (PVD/match), สิทธิบำนาญ (ประกันสังคม/กบข)

### สิ่งที่ต้องสรุป
1. **Net worth สำหรับเกษียณ** — แยก "สินทรัพย์ที่ใช้เกษียณได้จริง" (เงินลงทุน/กองทุนเกษียณ) ออกจากสินทรัพย์ที่สภาพคล่องต่ำ/ไม่ได้ตั้งใจขาย (เช่นบ้านที่อยู่เอง) — อย่านับรวมมั่ว
2. **Savings rate** = (ออม+ลงทุนต่อปี) / รายได้ต่อปี — ตัวชี้วัดที่ทรงพลังกว่ายอดเงินก้อน เพราะกำหนดทั้ง "ต้องสะสมอีกเท่าไร" และ "เคยชินกับรายจ่ายระดับไหน"
3. **Cashflow Priority (จัดลำดับก่อนคุย allocation/เกษียณ)** — แผนเกษียณพังบ่อยเพราะมองข้าม liquidity/หนี้/กำลังออมต่อเดือน ลำดับตั้งต้น (ปรับได้ตามข้อจำกัด เช่น รายได้ไม่นิ่ง/ภาระครอบครัว/ใกล้เกษียณ):
   1. สภาพคล่อง / เงินสำรองฉุกเฉิน
   2. หนี้ดอกเบี้ยสูง
   3. employer match / สวัสดิการที่การันตี (เงินที่ไม่ควรพลาด)
   4. เงินออมเพื่อเกษียณ (vehicle สิทธิภาษี)
   5. ลงทุนระยะยาวทั่วไป
   6. เป้าหมาย/ไลฟ์สไตล์เสริม
   ระบุเหตุผลของแต่ละลำดับ ถ้ายังไม่พร้อมขั้นต้น (1–2) ให้ flag ว่าจัดก่อน ไม่กระโดดไปคำนวณ corpus
4. **ช่องว่างข้อมูล** — อะไรขาดและกระทบ projection อย่างไร

### Output contract (BASE)
1. ตารางฐานะ: สินทรัพย์เกษียณได้จริง / หนี้ / เงินสำรอง / savings rate `[ข้อเท็จจริง]`/`[คำนวณ]`
2. **Cashflow priority order** (1–6) + เหตุผลของแต่ละลำดับ + ธงสิ่งที่ควรจัดก่อน
3. ข้อมูลที่ขาด + ต้องไปหาจากไหน
4. next-step hint → NUM (ไม่บังคับ)

## RSK — เช็กลิสต์ความเสี่ยงเกษียณ (ทำก่อนล็อกแผน)

จุดประสงค์: แผนเกษียณส่วนใหญ่พังเพราะความเสี่ยงที่มองข้าม ไม่ใช่เพราะคำนวณผิดทศนิยม — ไล่ให้ครบก่อนตัดสินใจ

ไล่ทุกหมวด ระบุระดับ (ต่ำ/กลาง/สูง/ไม่รู้) + ผลกระทบ + ทางลด:

| ความเสี่ยง | คำถามวินิจฉัย | ทางลดที่พบบ่อย |
|---|---|---|
| **Longevity** (อายุยืนกว่าที่คิด) | วางแผนถึงอายุเท่าไร? เผื่อ tail ถึง ~90–95+ หรือยัง? | วางแผนถึงช่วงอายุสูง, แหล่งรายได้ตลอดชีพ (บำนาญ/annuity) |
| **เงินเฟ้อ** | ใช้รายจ่าย real หรือ nominal? เผื่อค่าครองชีพโตระยะยาวไหม? | สินทรัพย์ที่โตชนะเงินเฟ้อ, ทบทวนแผนเป็นระยะ |
| **FX / สกุลเงิน** | รายจ่ายสกุลหนึ่ง (เช่น THB) แต่สินทรัพย์/ผลตอบแทนอีกสกุล (เช่น USD) ไหม? โมเดล liability ในสกุลที่ใช้จ่ายหรือยัง? | กันบางส่วนในสกุลที่ใช้จ่าย, ระบุ FX treatment ใน ledger (ดู SKILL.md), ทำ stress case |
| **Sequence-of-returns** | ถ้าตลาดตกหนักช่วง 2–3 ปีแรกหลังเกษียณ แผนยังรอดไหม? | cash/bond buffer, ถอนแบบยืดหยุ่น (ดู DRAW) |
| **ค่ารักษา/Long-term care** | เผื่อค่าสุขภาพที่โตเร็วกว่าเงินเฟ้อทั่วไป + ค่าดูแลระยะยาวไหม? | ประกันสุขภาพ/บำนาญสุขภาพ, กันเงินก้อนแยก |
| **รายได้กระจุกแหล่งเดียว** | พึ่งแหล่งเดียว (เช่นบำนาญรัฐ/ลูกหลาน/สินทรัพย์ก้อนเดียว) ไหม? | กระจายแหล่งรายได้หลังเกษียณ |
| **Liquidity/สภาพคล่อง** | สินทรัพย์ส่วนใหญ่ขายยาก/ติด lock ไหม? | กันสภาพคล่องพอสำหรับ 1–3 ปีของรายจ่าย |
| **กฎ/ภาษีเปลี่ยน** | แผนพึ่งสิทธิภาษี/วงเงินที่อาจถูกแก้ไหม? | ไม่พึ่งสิทธิเดียว, ทบทวนเมื่อกฎเปลี่ยน |
| **พฤติกรรม** | ถือผ่าน drawdown ใหญ่ได้ไหม? จะเผลอถอนก่อนกำหนดไหม? | ออกแบบให้ทำตามง่าย, อัตโนมัติ |
| **ครอบครัว/ภาระพึ่งพิง** | มีคนต้องดูแล (พ่อแม่/ลูก/คู่สมรส) ที่กระทบแผนไหม? | รวมภาระเข้า budget, แผนสำรอง |

**กฎ:** ถ้าหลายหมวด = สูง/ไม่รู้ → ประเด็นคือ "แผนเปราะ" ไม่ใช่ "ตัวเลข corpus" — สื่อให้ผู้ใช้เห็นว่าต้องอุดความเสี่ยงก่อน ไม่ใช่แค่ออมเพิ่ม

### Output contract (RSK)
1. ตารางความเสี่ยง + ระดับ + ทางลด
2. Top 3 ความเสี่ยงที่ควรจัดการก่อน + เหตุผล
3. ความเสี่ยงที่ "ไม่รู้" เพราะข้อมูลขาด (แยกชัด)
4. next-step → DEC หรือ DRAW

## Anti-patterns (BASE & RSK)
- นับบ้านที่อยู่อาศัยเองเป็น "เงินเกษียณ" โดยไม่มีแผนขาย/ปล่อยเช่าจริง
- มองแต่ยอดเงินก้อน ไม่ดู savings rate และพฤติกรรมรายจ่าย
- กระโดดคำนวณ corpus ทั้งที่ยังไม่มีเงินสำรอง/มีหนี้ดอกเบี้ยสูง
- ประเมินความเสี่ยงเป็น "มี/ไม่มี" แทนระดับ + ผลกระทบ + ทางลด
- ลืม longevity และ sequence risk (สองตัวที่ฆ่าแผนเกษียณบ่อยที่สุด)
- เหมารวมค่ารักษากับเงินเฟ้อทั่วไป ทั้งที่โตเร็วกว่า
